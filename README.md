# School-Grades-Management-System

Проць Христина, Лапчук Соломія, Левицький Микола, Дурко Ростислав

Система управління шкільними оцінками — це настільний додаток, призначений для зручного керування даними про учнів, предмети, вчителів та їхні оцінки.

Додаток дозволяє ефективно відстежувати успішність учнів, автоматизувати обчислення середніх балів, формувати звіти та аналізувати навчальні досягнення. Важливою особливістю є те, що система працює без використання бази даних — усі дані зберігаються локально у файлах, що робить її простою у використанні та налаштуванні.

##  Мета проєкту

Основною метою є спрощення роботи шкільного персоналу з навчальною інформацією, зменшення ручної роботи та підвищення точності обліку оцінок. Система допомагає:

* автоматично обчислювати середній бал,
* формувати рейтинги учнів,
* створювати зручні та зрозумілі табелі успішності,
* швидко знаходити необхідну інформацію.

##  Основні можливості

### 1. Управління учнями

* Додавання нових учнів (ім’я, прізвище, клас).
* Редагування та видалення інформації про учнів.
* Пошук за різними критеріями (ім’я, прізвище, клас).
* Збереження даних локально без необхідності підключення до бази даних.
* Можливість перегляду повної інформації про кожного учня.

### 2. Предмети та вчителі

* Створення та редагування списку предметів.
* Призначення вчителів до конкретних предметів.
* Додавання та редагування даних про вчителів.
* Реалізація логічної структури: **вчитель → предмет → клас**.
* Гнучке налаштування навчального процесу.

### 3. Журнал оцінок

* Внесення оцінок для кожного учня з конкретного предмета.
* Редагування та видалення оцінок.
* Автоматичний розрахунок:

  * середнього балу учня з предмета,
  * загального середнього балу по класу.
* Відображення оцінок у вигляді зручних таблиць (WPF).
* Можливість швидкого перегляду динаміки успішності.

### 4. Звіти та статистика

* Формування табелів успішності для учнів.
* Побудова рейтингу учнів у межах класу.
* Аналіз успішності за предметами та класами.
* Експорт звітів у формати:

  * CSV,
  * текстові файли (TXT).
* Підготовка даних для друку або подальшої обробки.

### 5. Локальне збереження даних

* Зберігання всієї інформації у файлах або в пам’яті програми.
* Відсутність потреби в сервері чи складній конфігурації.
* Просте резервне копіювання та відновлення даних.
* Висока швидкість роботи завдяки локальному доступу.

##  Використані технології

* **WPF (.NET)** — для створення сучасного, інтуїтивно зрозумілого інтерфейсу (таблиці, форми, візуальні компоненти).
* **C# (.NET)** — реалізація бізнес-логіки програми.
* **Файлова система / серіалізація** — для локального збереження даних.
* **Entity Framework (EF Core)** — для роботи з об’єктами та управління даними в пам’яті.
* **ADO.NET** — для демонстрації роботи із SQL-запитами та структурованими даними.

##  Додаткові переваги

* Простота використання навіть для користувачів без технічного досвіду.
* Зрозумілий та сучасний інтерфейс.
* Можливість масштабування функціоналу в майбутньому (наприклад, додавання бази даних або веб-версії).
* Підвищення ефективності роботи навчального закладу.
